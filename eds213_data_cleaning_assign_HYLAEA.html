Our goal is to:

  • Clean the Water_cover column to transform it into the correct data type and respect expectations for a percentage

  • Clean the Land_cover column to transform it into the correct data type and respect expectations for a percentage

  • Use the relationship that the three cover columns (Snow, Water, Land) should sum up to 100% to infer missing values where possible and recompute the Total_cover column as needed

Water_cover Cleaning

Exploration

We find the same symbols seen in Snow_cover that are not numeric. According to the metadata, “a blank or ‘-’ indicates absence of data”, so these values will be replaced with NA. Values such as n/a, unk, and . are not defined in the metadata and will be treated as NA for consistency.

Fix Non-numeric Values

# Replace unknown/missing string values with NA
cover_df <- cover_df %>%
  mutate(
    Water_cover = ifelse(
      Water_cover %in% c(".", "-", "n/a", "unk"), NA, Water_cover
    ))

Inferring Missing Values for the Total_cover column

Based on the metadata, the columns Snow_cover + Water_cover + Land_cover should always sum to 100. Where exactly one of the three columns is NA, we can infer it as 100 - sum(the other two). We cannot guess the values when two or more columns are missing.

Let’s first check how many rows fall into each case:

# Count rows by number of NAs across the three cover columns
cover_df %>%
  mutate(na_count = is.na(Snow_cover) + is.na(Water_cover) + is.na(Land_cover)) %>%
  count(na_count)
# A tibble: 4 × 2
  na_count     n
     <int> <int>
1        0 41968
2        1    96
3        2    54
4        3   712

There are 96 rows with exactly one NA that can be imputed. Rows with 2 or 3 missing values cannot be recovered.

# Impute missing values where exactly one cover column is NA
cover_df <- cover_df %>%
  mutate(
    # Count NAs per row
    na_count = is.na(Snow_cover) + is.na(Water_cover) + is.na(Land_cover),

    # Infer Snow_cover if it is the only missing value
    Snow_cover = ifelse(na_count == 1 & is.na(Snow_cover), 100 - Water_cover - Land_cover, Snow_cover),

    # Infer Water_cover if it is the only missing value
    Water_cover = ifelse(na_count == 1 & is.na(Water_cover), 100 - Snow_cover - Land_cover, Water_cover),

    # Infer Land_cover if it is the only missing value
    Land_cover = ifelse(na_count == 1 & is.na(Land_cover), 100 - Snow_cover - Water_cover, Land_cover)
  ) %>%
  # Drop the helper column before saving
  select(-na_count)

Check if it worked

Handle All-zero Rows

The rows where all three cover columns are 0 cannot represent a valid observation, since snow, water, and land together must cover 100% of the survey area. We consider that these rows are most likely unfilled fields rather than true measurements, so we set them to NA.

# Set all-zero rows to NA across all three cover columns
cover_df <- cover_df %>%
  mutate(
    all_zero = Snow_cover == 0 & Water_cover == 0 & Land_cover == 0,
    Snow_cover  = ifelse(all_zero, NA, Snow_cover),
    Water_cover = ifelse(all_zero, NA, Water_cover),
    Land_cover  = ifelse(all_zero, NA, Land_cover)
  ) %>%
  select(-all_zero)

Recompute Total_cover

Now, we recompute Total_cover from the cleaned columns. The rows where the three columns do not sum to 100 receive NA, because an explicit missing value is better than a wrong total. I chose to preserve the individual cover columns as-is because most of them can still be used independently for analysis even when the other columns are missing.

# Recompute Total_cover; set to NA if columns do not sum to 100
cover_df <- cover_df %>%
  mutate(
    Total_cover = Snow_cover + Water_cover + Land_cover,
    Total_cover = ifelse(Total_cover != 100, NA, Total_cover)
  )
